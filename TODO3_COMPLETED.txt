✅ Реализация эндпоинта перемещения в trash по текстовому запросу

Выполнены все требования из todo3.txt:

1. ✅ Создан эндпоинт POST /photos/move-to-trash
   - Принимает текстовый запрос (query)
   - Выполняет семантический поиск через CLIP эмбеддинги
   - Перемещает найденные файлы в папку trash

2. ✅ Параметр model (требование 1.1)
   - Добавлен опциональный параметр model
   - Поддерживаются модели: ViT-B/32, ViT-B/16, ViT-L/14, SigLIP
   - По умолчанию использует модель из настроек

3. ✅ Сохранение структуры папок (требование 1.2)
   - Файлы перемещаются с сохранением относительной структуры
   - Используется os.path.relpath(file_path, "/photos")
   - Создаются необходимые поддиректории в trash
   - Реализация идентична duplicate_finder._move_to_trash()

4. ✅ Индексы не удаляются (требование 1.3)
   - Записи в БД остаются нетронутыми
   - Удаляются только файлы с диска

Дополнительные возможности:
- Автоматический перевод запросов на английский
- Настраиваемый порог схожести (similarity_threshold, default 0.1)
- Ограничение количества файлов (top_k)
- Детальный отчет с указанием порога схожести
- Генерация отчетов в /reports/trash_moved_*.txt
- Критические проверки безопасности перед перемещением файлов

Безопасность:
- TRASH_DIR = "/.trash" (вне /photos, не переиндексируется)
- Проверка существования и доступности TRASH_DIR перед операциями
- HTTPException 500 если TRASH_DIR недоступна (предотвращает потерю данных)
- Logging всех операций перемещения
